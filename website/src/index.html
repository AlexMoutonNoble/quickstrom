WebCheck

Exploratory testing, automated.

Why use WebCheck?

  • Reduce testing effort

    Generate thousands of test cases instead of writing them manually. Lower the maintenance burden of your browser testing by writing concise specifications with looser coupling.

  • Find complex bugs

    WebCheck simulates complex and unexpected user behavior using generative random testing. When the specification is violated, WebCheck finds a minimal failing example.

  • Understand your system

    Focus on specifying your system, not on writing test cases. A formal specification lets you run WebCheck, but can also increase your team's understanding of the system.

  • Adopt gradually

    WebCheck works with any web application that renders DOM elements. Start simple, and gradually refine your specification to increase coverage and confidence.

I'm interested!

Great! WebCheck is currently in the making. Early prototypes showed very promising results, and a simple specification found inconsistencies in two mainstream implementations of TodoMVC. The upcoming implementation features a PureScript-based specification language, inspired by TLA+.

The first publicly available version will likely be a SaaS product, possibly with a free version for open-source projects. Longer-term plans include an integrated online editor for specifications, powerful debugging and introspection tools for troubleshooting failed tests, and tight integration with GitHub and CI services.
